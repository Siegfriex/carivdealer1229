## 부록 A. FRD(기능-화면-플로우-지표 매핑) — `기술설계서.md` 범위

### A.0 범위(검증 범위/제외 범위)

- **검증 범위**: 본 문서에 서술된 **해외 바이어용 웹 서비스(Cariv Buyer Front)** + **문서 자동화(Document Hub/AutoExport)** 기능
- **제외 범위**: 딜러/검차/탁송/정산의 상세 FRD는 `PRD_Phase1_2025-12-31 copy.md` 및 `통합_PRD.md`의 레지스트리를 참조
- **정합 기준**: ID 체계(FUNC/SCR/FLOW/KPI)와 매핑 규칙은 `통합_PRD.md` v1.1(2025-12-23)과 동일하게 유지

### A.1 관계 규칙(개발자 기준)

- **SCR(화면) → FUNC(기능)**: 화면은 1개 이상의 기능을 제공한다.
- **FUNC(기능) → SCR(화면)**: 사용자 액션 기반 기능은 1개 이상의 화면에 매핑된다. (시스템 자동 기능은 `SCR = -` 가능)
- **FLOW(플로우)**: 여러 화면(SCR)의 순서/상태 전이로 구성되며, 기능(FUNC)은 최소 1개 이상의 FLOW에 연결된다.
- **KPI**: KPI는 최소 1개 이상의 기능(FUNC)에 연결되어 측정 이벤트(로그/메트릭)가 정의되어야 한다.
