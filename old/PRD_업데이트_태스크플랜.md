### Task 1.2: FLOW-03 매핑 테이블 수정
**우선순위**: Critical  
**예상 소요 시간**: 1시간  
**담당**: 개발팀

**작업 내용**:
- Section 7.2 "FLOW → Screen 매핑" 테이블 수정
- FLOW-03의 화면 순서를 실제 플로우 순서에 맞게 재정렬
- FLOW-02의 화면(SCR-0201, SCR-0202)을 FLOW-03에 포함 명시

**수정 전**:
```
FLOW-03: SCR-0101 → SCR-0200 → SCR-0300 → SCR-0400 → SCR-0102 → SCR-0103
```

**수정 후**:
```
FLOW-03: SCR-0200 (차량 등록) → [FLOW-02 연결] SCR-0201 (검차 신청) → SCR-0202 (검차 결과) → SCR-0300 (판매 방식 선택) → SCR-0400 (경매) 또는 SCR-0102 (일반 판매) → SCR-0103 (판매 내역)
```

**수정 위치**: PRD Section 7.2  
**검증 기준**: Section 9.3 FLOW-03 태스크 플로우와 일치

---

### Task 1.3: 사이트맵 구조 수정
**우선순위**: Critical  
**예상 소요 시간**: 2시간  
**담당**: 기획팀

**작업 내용**:
- Section 8.2 "딜러 사이트맵" 구조 수정
- "차량 등록 > 검차 신청" 구조를 분리
- 실제 플로우 구조에 맞게 재구성

**수정 전**:
```
차량 관리 > 차량 등록 > 등록원부 OCR (SCR-0200)
차량 관리 > 차량 등록 > 검차 신청 (SCR-0201)
```

**수정 후**:
```
차량 관리 > 차량 등록 (SCR-0200)
검차 관리 > 검차 신청 (SCR-0201)
검차 관리 > 검차 결과 조회 (SCR-0202)
차량 관리 > 차량 목록 (SCR-0101)
차량 관리 > 판매 방식 선택 (SCR-0300)
```

**수정 위치**: PRD Section 8.2  
**검증 기준**: Section 7.2 FLOW → Screen 매핑과 일치

## 3. output.json 세부 요소 반영 (중기 개선)

### Task 3.1: IA 항목을 PRD 부록에 추가
**우선순위**: Medium  
**예상 소요 시간**: 4시간  
**담당**: 기획팀

**작업 내용**:
- Section 23 부록에 "IA 1.0 상세 항목" 섹션 추가
- output.json의 IA 항목을 PRD 형식으로 변환하여 추가

**추가 내용**:

- 각 항목의 기능 정의, 화면 ID, Depth 구조
- PRD Function/Screen과의 매핑 관계

**수정 위치**: PRD Section 23 부록  
**검증 기준**: 모든 IA 항목이 문서화되고 매핑 관계 명시

### Task 3.2: SPM 정책을 PRD Decision/NFR에 반영
**우선순위**: Medium  
**예상 소요 시간**: 3시간  
**담당**: 기획팀

**작업 내용**:
- output.json의 SPM 정책을 PRD Decision Log 또는 NFR에 반영
- 매핑 가능한 정책은 Decision/NFR로 변환
- 매핑 불가한 정책은 GAP Log에 추가

**추가 내용**:
- 새로운 Decision 항목 추가 (필요시)
- 새로운 NFR 항목 추가 (필요시)
- GAP Log에 미반영 정책 추가

**수정 위치**: PRD Section 19 Decision Log, Section 12 NFR, Section 20 GAP Log  
**검증 기준**: 모든 SPM 정책이 Decision/NFR/GAP 중 하나에 매핑
