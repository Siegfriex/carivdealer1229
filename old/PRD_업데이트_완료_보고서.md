## 2. Critical Issues 해결 내역

### 2.1 상태 전이-화면 매핑 테이블 수정 ✅

**문제점**:
- Draft 상태가 "SCR-0200-Draft"로 표시되지만 Screen Registry에는 존재하지 않음
- Inspecting 상태가 "SCR-0201-Progress"로 표시되지만 Screen Registry에는 존재하지 않음

**해결 내용**:
- Section 15.1 "차량 상태별 화면 매핑" 테이블 수정
- `SCR-0200-Draft` → `SCR-0200` (접미사 제거)
- `SCR-0201-Progress` → `SCR-0201` (접미사 제거)
- 주석 추가: 상태별 화면 매핑은 Screen Registry의 실제 Screen ID를 사용

**수정 위치**: PRD Section 15.1  
**검증 결과**: ✅ Screen Registry와 일치 확인

---

### 2.2 FLOW-03 매핑 테이블 수정 ✅

**문제점**:
- FLOW-03 매핑 테이블의 화면 순서가 실제 플로우 순서와 불일치
- FLOW-02의 화면(SCR-0201, SCR-0202)이 FLOW-03 플로우에 포함되어야 하나 매핑 테이블에는 없음

**해결 내용**:
- Section 7.2 "FLOW → Screen 매핑" 테이블 수정
- FLOW-03의 화면 순서를 실제 플로우 순서에 맞게 재정렬:
  ```
  1. SCR-0200 (차량 등록)
  2. SCR-0201 (검차 신청) - FLOW-02 연결
  3. SCR-0202 (검차 결과 조회) - FLOW-02 연결
  4. SCR-0300 (판매 방식 선택)
  5. SCR-0400 (경매 상세) 또는 SCR-0102 (일반 판매 제안 목록)
  6. SCR-0103 (판매 내역)
  ```
- SCR-0101(차량 목록)을 순서 0으로 표시 (조회용, 플로우 시작점 아님)
- 주석 추가: FLOW-02와 FLOW-03의 관계 명확화

**수정 위치**: PRD Section 7.2  
**검증 결과**: ✅ Section 9.3 FLOW-03 태스크 플로우와 일치 확인

---

### 2.3 사이트맵 구조 수정 ✅

**문제점**:
- 사이트맵의 "차량 등록 > 검차 신청"이 3depths로 표시되지만, 실제로는 별도 화면으로 분리
- "검차 신청"이 두 곳에 중복 표시됨
- "탁송 예약"이 일정 선택과 배차 확정으로 분리되어 있지만 같은 화면 사용

**해결 내용**:
- Section 8.2 "딜러 사이트맵" 구조 수정
- "차량 등록 > 검차 신청" 구조를 분리:
  ```
  차량 관리 > 차량 등록 (SCR-0200)
  검차 관리 > 검차 신청 (SCR-0201)
  검차 관리 > 검차 결과 조회 (SCR-0202)
  ```
- 중복 항목 제거:
  - "검차 신청"을 "검차 관리"로만 이동
  - "차량 관리 > 차량 등록"에서 "검차 신청" 제거
- "탁송 예약" 구조 통합:
  ```
  탁송 관리 > 탁송 예약/배차 (SCR-0600) - 통합
  탁송 관리 > 탁송 내역 (SCR-0601)
  탁송 관리 > 인계 승인 (SCR-0601)
  ```

**수정 위치**: PRD Section 8.2  
**검증 결과**: ✅ Section 7.2 FLOW → Screen 매핑과 일치 확인
