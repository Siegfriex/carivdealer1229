### 3.2 사용 가이드 핵심 내용

**개발 시 활용 방법**:
1. 세부 기능 구현 시: output.json의 IA 항목을 참조하여 UI/UX 세부 사항 확인
2. 테스트 케이스 작성 시: IA 항목별로 테스트 케이스 작성 가능
3. 정책 구현 시: SPM 정책을 참조하여 비즈니스 규칙 확인
4. Decision Log 확인: 매핑된 Decision ID를 통해 관련 결정사항 확인
5. NFR 확인: 매핑된 NFR ID를 통해 비기능 요구사항 확인

**주의사항**:
- output.json의 항목은 **참조 자료**이며, PRD의 Function/Decision/NFR이 **최종 명세**입니다
- 두 문서 간 불일치가 있을 경우, **PRD를 우선**합니다
- 매핑 불가한 항목은 Phase 2 이후 검토 대상입니다

## 6. 개발자 활용 가이드

### 6.1 output.json을 언제 참조해야 하나요?

1. **UI/UX 세부 구현 시**: output.json의 IA 항목을 참조하여 화면별 세부 기능 확인
2. **비즈니스 규칙 구현 시**: SPM 정책을 참조하여 정책 로직 확인
3. **테스트 케이스 작성 시**: IA 항목별로 테스트 케이스 작성
4. **문서화 시**: IA 항목을 참조하여 기능 명세 보완

### 6.2 PRD와 output.json 중 어느 것을 우선해야 하나요?

**답: PRD를 우선합니다.**

- PRD는 **SSOT (Single Source of Truth)**입니다
- output.json은 **참조 자료**입니다
- 두 문서 간 불일치가 있을 경우, PRD를 기준으로 합니다
- output.json의 세부 항목이 PRD에 반영되지 않은 경우, Phase 2 이후 검토 대상입니다

### 6.3 매핑 불가 항목은 어떻게 처리하나요?

- 매핑 불가 항목은 Phase 2 이후 검토 대상입니다
- 개발 시 해당 항목이 필요하다면, PRD에 Function/Decision/NFR 추가를 요청하세요
- 임시로 output.json을 참조하여 개발할 수 있으나, PRD 업데이트가 필요합니다
