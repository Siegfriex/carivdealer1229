# Google AI Studio 에이전트 사전 지침 (요약본)

## 🚨 절대 준수 사항

1. **기존 파일 보존 최우선**
   - ❌ 기존 파일(index.html, index.tsx 등)을 전체 재작성 금지
   - ✅ 기존 파일을 먼저 읽고 필요한 부분만 수정
   - ✅ 작은 단위로 점진적 수정

2. **PRD 문서 우선 참조**
   - SSOT: `FOWARDMAX/PRD_Phase1_2025-12-31.md` (v2.13)
   - 모든 작업 전 PRD 문서 확인 필수
   - Function/Screen/Flow/API 구현 시 PRD Section 6~14 참조

3. **작업 절차**
   - Step 1: 기존 파일 읽기 및 분석
   - Step 2: PRD 문서에서 관련 명세 확인
   - Step 3: 필요한 부분만 수정 (전체 재작성 금지)
   - Step 4: 수정 후 검증

4. **금지 사항**
   - ❌ 전체 파일 재작성
   - ❌ PRD 문서 무시
   - ❌ 기존 코드 구조 완전 변경
   - ❌ 불필요한 새 파일 생성

5. **권장 사항**
   - ✅ 한 번에 하나의 기능/화면/컴포넌트만 수정
   - ✅ 기존 코드 스타일/구조/네이밍 유지
   - ✅ PRD 명세에 맞게 구현
   - ✅ 기존 컴포넌트 재사용 우선

## 📋 작업 시작 전 체크리스트

- [ ] 사용자 요청 범위 정확히 이해
- [ ] 기존 파일 읽고 분석 완료
- [ ] PRD 문서에서 관련 명세 확인 완료
- [ ] 수정 필요 부분만 정확히 식별
- [ ] 기존 기능에 영향 없음 확인
- [ ] 점진적 수정 가능 여부 확인

## 📚 PRD 문서 참조 가이드

- Section 6: ID 레지스트리 (Screen, Function, Flow, API 등)
- Section 7: 기능-화면-메타데이터 매핑
- Section 8: 정보구조(IA) 및 사이트맵
- Section 9: 유저플로우
- Section 10: FRD (세부 기능 명세)
- Section 12: NFR (비기능 요구사항)
- Section 13: TSD (기술 설계)
- Section 14: SRD (서비스 요구)
- Section 23.5~23.6: output.json 참조

## ⚠️ 자주 발생하는 실수 방지

1. "index.tsx를 처음부터 다시 작성" → ❌ 금지
2. "일반적인 React 패턴으로 구현" → ❌ PRD 우선
3. "더 나은 방법으로 다시 작성" → ❌ 기존 코드 보존
4. "새로운 구조로 전체 재구성" → ❌ 기존 구조 유지
