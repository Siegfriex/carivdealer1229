# ForwardMax 프론트엔드 개발 서버 실행 가이드

## 🚀 빠른 시작

### 방법 1: 배치 파일 사용 (가장 간단)

```batch
dev_start.bat
```

배치 파일이 자동으로:
1. Node.js 설치 확인
2. 의존성 설치 (필요한 경우)
3. 개발 서버 실행

### 방법 2: 수동 실행

```powershell
# 1. 프로젝트 디렉토리로 이동
cd C:\carivdealer

# 2. Node.js 설치 확인
node --version
npm --version

# 3. 의존성 설치 (처음 한 번만)
npm install

# 4. 개발 서버 실행
npm run dev
```

## 📋 사전 요구사항

### Node.js 설치

현재 시스템에 Node.js가 설치되어 있지 않거나 PATH에 등록되지 않은 것으로 보입니다.

**설치 방법**:
1. Node.js 공식 사이트 방문: https://nodejs.org/
2. **LTS 버전** 다운로드 (v20.x 이상 권장)
3. 설치 프로그램 실행 및 기본 설정으로 설치
4. **중요**: 설치 후 PowerShell/터미널을 완전히 종료하고 다시 시작

**설치 확인**:
```powershell
node --version  # v20.x.x 이상이어야 함
npm --version   # 10.x.x 이상이어야 함
```

## 🔑 환경 변수 설정

개발 서버 실행 전 `.env.local` 파일을 생성하세요:

```powershell
# .env.local.example을 복사하여 .env.local 생성
copy .env.local.example .env.local
```

`.env.local` 파일에 Gemini API 키를 입력:
```env
GEMINI_API_KEY=your_actual_api_key_here
```

## 🛠️ 문제 해결

### 문제 1: npm이 인식되지 않음

**증상**: `'npm'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다.`

**해결 방법**:
1. Node.js 재설치: https://nodejs.org/
2. 설치 후 **PowerShell/터미널 완전히 재시작**
3. 환경 변수 PATH에 Node.js 경로가 포함되어 있는지 확인

### 문제 2: 포트 3000이 이미 사용 중

**해결 방법**: `vite.config.ts`에서 포트 변경

```typescript
server: {
  port: 3001, // 다른 포트로 변경
}
```

### 문제 3: 의존성 설치 오류

**해결 방법**:
```powershell
# 캐시 클리어 후 재설치
npm cache clean --force
npm install
```

### 문제 4: GEMINI_API_KEY 오류

**해결 방법**:
1. `.env.local` 파일 생성 확인
2. 파일 내용에 `GEMINI_API_KEY=your_key` 형식으로 입력
3. 개발 서버 재시작
